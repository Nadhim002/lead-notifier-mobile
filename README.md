# Lead Notifier — Mobile App

The phone half of the IndiaMART Lead Notifier system. This Expo / React Native (Android-first) app
signs in with Google, registers the device, and listens for new leads that the
[Chrome extension](#related-repos--wire-contract) purchases on your PC — then alerts you instantly,
either as a **banner notification** or as a full-screen **incoming-call-style** takeover.

You sign in with the **same Google account** on your phone and on the extension. The extension writes
each purchased lead to Firebase and sends an Expo push; this app receives it and shows the alert even
when the app is backgrounded or killed.

---

## How it works (30-second version)

```
Extension (PC)  ──Expo push + Firebase write──▶  this app:
                                                    │
   signed in with the same Google account ──────────┤
                                                    ├─ headsup  → banner notification
                                                    └─ phonecall → full-screen "incoming lead" screen
                                                                   (native full-screen intent)
```

For the full picture — the app wiring, the four ways a lead can enter the UI, the auth seam, and the
Firebase data model — read [docs/ARCHITECTURE.md](docs/ARCHITECTURE.md). For domain terms see
[CONTEXT.md](CONTEXT.md).

## Repo layout

```
index.ts                 Expo entry (registerRootComponent)
App.tsx                  root: AuthProvider + navigation + 3 of the 4 notification-entry paths
                         (the live listener is in useLeadListener; see docs/ARCHITECTURE.md)
navigation.ts            navigation ref + navigateToIncomingLead helper
firebase.ts              Firebase app/auth/db singletons
notifications.ts         permissions, channels, Expo token, fireLeadNotification, leadNotificationText
deviceIdentity.ts        stable device id + devices/{uid}/{deviceId} ref (one source of truth)
channels.ts              Android channel IDs (must match the extension)
logger.ts                tagged console loggers (AppLog, AuthLog, NotifLog, LeadsLog, DeviceLog…)
hooks/
  AuthProvider.tsx       the single auth session (one onAuthStateChanged) + useAuth()
  useDeviceRegistration.ts   registers this device in Firebase
  useNotificationStyle.ts    banner vs phonecall preference (persisted + synced)
  useLeadListener.ts     subscribes to leads/{uid}/new and fires the alert
screens/                 SignIn · Home · IncomingLead · Settings
modules/PhonecallNotification.ts   JS bridge to the native full-screen-intent module
plugins/withFullScreenIntent.ts    Expo config plugin that writes the Kotlin native module at prebuild
docs/                    ARCHITECTURE.md
```

## Prerequisites

- **Node.js 18+**
- **JDK 17** and **Android Studio** + Android SDK (see [BUILD_APK.md](BUILD_APK.md) for the full list)
- An Android device or emulator
- A **custom dev build** — **Expo Go will not work** (it can't receive Firebase/FCM notifications).
  See [DEBUG.md](DEBUG.md).

> ⚠️ **Read [AGENTS.md](AGENTS.md) before writing code.** This project runs unusually high versions
> (Expo SDK 56 / React Native 0.85 / React 19). Check the versioned Expo docs
> (`https://docs.expo.dev/versions/v56.0.0/`) rather than assuming current-stable API shapes.

```bash
npm install
```

## Configuration

| What | Where |
|---|---|
| Firebase config (same project as the extension) | `firebase.ts` + `google-services.json` |
| Google **web client id** (for Google sign-in) | `hooks/AuthProvider.tsx` (`WEB_CLIENT_ID`) |
| Expo push **projectId** | `app.json` (`extra.eas.projectId`) + `notifications.ts` |
| Notification channel IDs (must match the extension) | `channels.ts` |
| Native full-screen-intent module | generated by `plugins/withFullScreenIntent.ts` at prebuild |

## Notification styles

Set in **Settings**, stored per device in Firebase:

- **Banner** (`headsup`) — a standard heads-up notification on the `lead-alerts-banner` channel.
- **Phone Call** (`phonecall`) — the extension sends a **data-only** push; the native
  `LeadNotificationService` (from `plugins/withFullScreenIntent.ts`) fires a **full-screen intent**
  that opens the "incoming lead" screen over the lock screen. Android 14+ requires an explicit
  full-screen-intent permission grant, requested when you pick this style.

## Related repos & wire contract

The other half of this system is the PC client:

- **Extension:** [`indiamart-extension`](https://github.com/Nadhim002/indiamart-extension)

The two apps are **separate repos** sharing three formats by convention (not shared code). Change any
of these here and change them in the extension too:

| Shared format | Mobile source | Extension source |
|---|---|---|
| **Notification channel IDs** | `channels.ts` | `src/shared/channels.ts` |
| **Firebase project** (same RTDB) | `firebase.ts` | `src/shared/firebaseConfig.ts` |
| **Expo push payload shape** | consumed in `notifications.ts` / `useLeadListener.ts` | `src/shared/pushPayload.ts` |

A channel-ID mismatch makes Android silently drop the notification. See
[docs/ARCHITECTURE.md](docs/ARCHITECTURE.md#cross-repo-wire-contract).

## Security notes

- `google-services.json` and the Android signing keystores (`*.jks`) are sensitive. `.gitignore`
  covers `*.jks`, but `google-services.json` is currently tracked — treat it as project config, not a
  public secret, and keep it out of any public fork.
- The Firebase config in `firebase.ts` is the web/client config; access is governed by your Realtime
  Database security rules (not in this repo).
